## Что принести в разговор

- **Planogram — существует готовый формат**: профессиональный термин для схем расстановки товаров. Есть ли смысл посмотреть готовые planogram tools (RetailZoom, Apollo Retail) для вдохновения?
- **Настройка стеллажа — это разовая операция**: пользователь один раз настроил, потом годами не трогает. Значит, UX конструктора не обязан быть идеальным — важнее, чтобы он был надёжным и не допускал ошибок.
- **Import из Excel/CSV**: если конфигурация стеллажа сложная, проще ввести в таблицу и импортировать, чем кликать в UI.

## Ты — критик: оспорь возможные решения

1. **Визуальный drag-and-drop редактор — нужен ли он?**
   Это сложнейший UI. Разработка займёт значительное время. А пользователь использует его один раз при первом запуске. Может, текстовая конфигурация или простой мастер ("введи размеры ячеек: A1: 600×400, A2: 300×400...") надёжнее и быстрее в реализации?

2. **Разделение ячейки touch-жестом — реально ли?**
   Зажать, перетащить разделитель на маленьком экране — сложно. Особенно для неопытных пользователей. Может быть лучше: выбрать ячейку → кнопка "Разделить" → выбрать направление и количество частей.

3. **Конструктор vs. чтение стеллажа**: это разные режимы или один? Нельзя ли "настройка" и "просмотр остатков" выглядеть одинаково, просто в разных режимах?
