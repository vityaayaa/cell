## Что принести в разговор

Это финальная сессия. Принеси **полную карту экранов** — список всего, что было решено в S11–S13, плюс то, что могло быть упущено:

- **Экран каталога**: как сотрудник находит/редактирует товар? Это только admin или все? (S07)
- **Экран истории**: список заявок с датами, фильтром (S08)
- **Экран настроек**: кто-то должен менять роли, настраивать приложение. Где это живёт?
- **"Главный экран"**: что видит сотрудник при открытии? Сразу стеллаж? Это ключевое решение.

Дополнительно:
- **Онбординг при первом запуске**: стеллаж пустой, каталог пустой — что показать? Пустой экран отталкивает. Нужен мастер первого запуска.
- **Обработка ошибок**: что видит пользователь при ошибке сохранения (офлайн, нет памяти)?
- **Loading states**: синхронизация началась — что на экране?
- **Система уведомлений**: нужны ли push-уведомления? "Заявка не синхронизировалась 3 дня" — это важно?

## Ты — критик: оспорь возможные решения

1. **Нижний таббар с 4-5 вкладками — правильно ли?**
   Посчитай реальное количество top-level экранов для сотрудника: Стеллаж, Заявка, Чеклист, История. Для admin добавляется: Каталог, Настройки. Это 6 вкладок — уже много для таббара. Может, две отдельные навигационные структуры (по роли)?

2. **"Главный экран = стеллаж" — правильно ли?**
   Для сотрудника главное — начать обход. Для менеджера главное — посмотреть сводку по остаткам. Один главный экран не может быть оптимальным для обоих.

3. **Общий визуальный стиль — какой?**
   Строительный магазин: грубость, практичность, рабочий контекст. Но приложение на телефоне — нормальный мобильный UX. Напоминать об "индустриальном" стиле (оранжевый, жёлтый, тёмный) или нейтральный минимализм (белое, серое, акцент-цвет)?
