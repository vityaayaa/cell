# S13 — UI: Заявка и чеклист

## Контекст использования

Два последовательных экрана одной сессии:
- **Черновик заявки** (`ordering`) — сотрудник или admin редактирует список до финализации
- **Чеклист** (`fulfilling`) — сотрудник отмечает взятые позиции на складе

Переход `ordering → fulfilling` необратим (S05). Оба экрана — телефон, одна рука, стоя.

---

## Переходы между экранами

```
[Обход S12] ──→ [Черновик заявки] ──→ [Чеклист] ──→ [История]
                      │                    │
               ← Вернуться к обходу    (нельзя назад)
```

| Переход | Как | Куда |
|---------|-----|------|
| Обход → Черновик | Кнопка "→ К заявке" (S12) | Экран ordering |
| Черновик → Обход | Кнопка "← К обходу" в header | Экран sweeping |
| Черновик → Чеклист | Финализация (необратимо) | Экран fulfilling |
| Чеклист → Сессии | Header ← | Экран Сессий (S14) |

---

## Экран черновика заявки (ordering)

### Структура экрана

```
← К обходу                            Заявка

────────────────────────────────────────────
  Брусок 50×50×3000          4 пачки · 16 шт
  Доска 20×100×6000           2 пачки · 8 шт
  Труба ПВХ ⌀110×3000         3 пачки · 6 шт
  ...

────────────────────────────────────────────
  Пограничные позиции
  Штапик                 дефицит 3 шт < пачки
  Плинтус 30×10×3000     дефицит 2 шт < пачки

────────────────────────────────────────────

              [Финализировать заявку]
```

- **Список строк** — сортировка: материал (выбранный первым) → длина убывает → название А→Я (S05)
- **Блок пограничных позиций** — отдельно внизу, заголовок "Пограничные позиции"
- **Ячейки без обновлённого остатка** — не включаются в заявку (S05)
- **Кнопка "← К обходу"** — в header слева, возвращает в `sweeping` без потери данных

---

### Диалог строки заявки

Тап на любую строку основного списка → bottom sheet:

```
Брусок 50×50×3000
Дефицит: 30 шт · расчётное 2 пачки

[−]  [ 2 ]  [+]
              [Сохранить]

────────────────────────────
[Удалить из заявки]
```

| Элемент | Детали |
|---------|--------|
| Заголовок | Полное имя товара |
| Инфо | Дефицит в штуках + расчётное число пачек (только чтение) |
| `[−] [N] [+]` | Предзаполнено расчётным `quantity_packs`; прямой ввод числа |
| `[Сохранить]` | 56px, `--primary`. Активна всегда |
| `[Удалить из заявки]` | Красный текст, внизу. Удаляет строку из заявки без дополнительного подтверждения |

---

### Диалог пограничной строки

Тап на строку в блоке "Пограничные позиции" → bottom sheet:

```
Штапик
дефицит 3 шт — меньше одной пачки (4 шт)

[Включить в заявку]
```

Нажатие `[Включить в заявку]` → строка переходит в основной список + открывается стандартный диалог строки (редактирование количества + удаление).

---

### Финализация заявки

**Кнопка `[Финализировать заявку]`** — 48px (намеренно менее заметная согласно S00 для опасных действий), внизу экрана, не рядом с частыми действиями.

Нажатие → bottom sheet:

```
Готово к финализации

8 позиций · 24 пачки

Это нельзя отменить. Заявка будет
зафиксирована.

[Отмена]          [Финализировать →]
```

| Элемент | Детали |
|---------|--------|
| Итог | N позиций · M пачек — финальный чекпойнт |
| `[Отмена]` | Закрывает bottom sheet, возврат к списку |
| `[Финализировать →]` | 56px, `--primary`. Необратимый переход в `fulfilling` |

Защита двухшаговая: первое нажатие открывает sheet, второе — кнопка в другой точке экрана.

## Экран чеклиста (fulfilling)

### Структура экрана

```
←                          Чеклист   🖨

  Взято 3 из 8  ████████░░░░░░░░░░░
────────────────────────────────────────

  Брусок 50×50×3000                [Взял]
  4 пачки · 16 шт

  Доска 20×100×6000                [Взял]
  2 пачки · 8 шт

  ...

  ── Взятые и отсутствующие ──

  ✓  Труба ПВХ ⌀110×3000    Взял 3 пачки
     3 пачки · 6 шт

  ✗  Штапик                 Нет на складе
     1 пачка
```

- **Sticky header** "Взято N из M" — всегда виден, не скрывается при скролле. Прогресс-полоска (не убывающая, а заполняющаяся по мере отметки).
- **Иконка принтера 🖨** — в header справа. Запускает печать браузера (S06/S10).
- **Сортировка** — `pending` строки сверху в стандартной сортировке S05; `done`/`unavailable` уходят в конец отдельным блоком.

---

### Строка `pending`

```
┌────────────────────────────────────────────┐
│  Брусок 50×50×3000              [Взял]     │
│  4 пачки · 16 шт                           │
└────────────────────────────────────────────┘
```

| Элемент | Детали |
|---------|--------|
| Название + параметры | 16px, `--foreground` |
| Количество | 14px, `--muted-foreground` |
| `[Взял]` | 56px, `--primary` (оранжевый), справа |

---

### Механизм отметки

**Быстрое действие — кнопка `[Взял]` справа:**
- Один тап → `done`, `actual_packs = quantity_packs`
- Строка анимируется: fade + slide вниз, 150мс (S00) → уходит в конец списка

**Нестандартные случаи — тап на строку (не на кнопку):**

Bottom sheet:
```
Брусок 50×50×3000 · 4 пачки

[Взял всё           ]   ← зелёный (#10B981)
[Взял меньше...     ]   ← neutral
[Нет на складе      ]   ← muted / красный
```

`[Взял всё]` — то же что кнопка `[Взял]`: `done`, `actual_packs = quantity_packs`.

`[Нет на складе]` — `unavailable`, `actual_packs` не заполняется.

`[Взял меньше...]` → вложенный bottom sheet:

```
Брусок 50×50×3000

Сколько взял?
[−]  [ 4 ]  [+]    ← предзаполнено quantity_packs

              [Подтвердить]
```

Минимум — 1 пачка. Максимум — `quantity_packs`. Прямой ввод числа тоже работает.  
После подтверждения: `done`, `actual_packs` = введённое число.

---

### Строки после отметки

**`done`:**
```
┌────────────────────────────────────────────┐
│ ✓  Брусок 50×50×3000      Взял 4 пачки    │
│    4 пачки · 16 шт                         │
└────────────────────────────────────────────┘
```
Фон: зелёный tint (`#10B981` с 10% opacity). Иконка `Check` (Lucide). Текст muted.  
Если `actual_packs < quantity_packs`: "Взял 3 из 4 пачек".

**`unavailable`:**
```
┌────────────────────────────────────────────┐
│ ✗  Брусок 50×50×3000    Нет на складе     │
│    4 пачки · 16 шт                         │
└────────────────────────────────────────────┘
```
Фон: `--muted`. Иконка `X` (Lucide). Текст muted.

Оба состояния различимы без цвета: иконка `✓` / `✗` + текстовая подпись (S00/A3).

---

### Undo (исправление ошибки)

Любой статус обратим до перехода сессии в `completed` (S06).

Тап на строку `done` или `unavailable` → тот же bottom sheet с тремя действиями. Сотрудник выбирает нужный статус.

Отдельного toast-undo нет — ошибка заметна сразу, исправление в одно действие.

---

### Завершение сессии

Когда все строки в статусе `done` или `unavailable` — сессия автоматически переходит в `completed` (S05).

Toast в нижней части экрана:
```
✓ Все позиции отмечены
▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░  ← убывающий прогресс-бар (S00)
```

Toast автозакрывается (~3 сек). Сотрудник остаётся на экране чеклиста. Переход к истории — через экран «Сессии» (header ← или вкладка для admin).

---

### PDF-экспорт

Иконка `Printer` (Lucide) в header → вызов `window.print()`. Браузер открывает диалог печати.

Бланк (S06): та же сортировка что в чеклисте на момент печати (`done`/`unavailable` в конце). Поля: название товара, количество пачек, ячейка для галочки "взял", крестика "нет на складе", строка "взял столько: ___".

## Принятые решения

| Решение | Суть | Почему |
|---------|------|--------|
| Кнопка `[Взял]` + тап на строку | Два уровня действий | Быстрый случай — один тап; нестандартные — через bottom sheet без риска случайного нажатия |
| Свайп не используется | Кнопка вместо свайпа | Складские условия: одна рука, движение, риск случайного свайпа при прокрутке. Кнопка надёжнее |
| Диалог строки заявки — bottom sheet | Тап на строку → редактирование + удаление | Защита от случайного редактирования при прокрутке (inline edit опасен) |
| Пограничная позиция — двухшаговое включение | `[Включить в заявку]` → стандартный диалог | Явное намерение включить пограничную позицию; потом сразу настройка количества |
| Финализация — bottom sheet с итогом | 48px кнопка → подтверждение в другом месте | Двухшаговая защита; итог (N позиций, M пачек) — финальный чекпойнт перед необратимым переходом |
| Анимация строки — сразу вниз | Нет паузы "показать ✓ на месте" | Мгновенная реакция (S00/A4); прогресс-строка обновляется сразу |
| Toast завершения — auto-dismiss | Нет action-кнопки в тосте | История доступна через экран Сессий; принудительный переход не нужен |
| Undo через повторный тап | Нет отдельного undo-toast | 10–20 позиций — ошибка заметна сразу; тап на строку = исправление |
| Просмотр заявки из чеклиста — не нужен | Данные в строках достаточны | Каждая строка показывает название и `quantity_packs`; отдельный экран заявки избыточен |

---

## Что намеренно за бортом

| Что | Почему |
|-----|--------|
| Свайп для отметки | Ненадёжен в рабочих условиях (одна рука, перчатки, движение) |
| Фильтр по материалу/длине | 10–20 позиций; сортировка уже группирует по материалу (S06) |
| Ссылка на заявку из чеклиста | Заявка locked; нужные данные уже в строках чеклиста |
| Автоматический переход при завершении | Сотрудник может хотеть просмотреть список; история доступна через Сессии |
| Отдельный экран итога после финализации | Bottom sheet с итогом достаточен |
| Hold-to-confirm для финализации | Bottom sheet понятнее для редкого действия |
| Групповое завершение (отметить всё сразу) | Нет реального сценария; каждая позиция требует физического действия на складе |
