# S01 — Каталог товаров

## Принятые решения

### Любое отличие = разный товар
Нет вариантов и шаблонов. Каждый SKU — отдельная запись. Доска 50×100×3000 и доска 50×100×6000 — два разных товара.

### Имя + параметры = идентификатор
Артикулов, штрихкодов, кодов поставщика нет. Каталог строится с нуля. Отображение = `name` + параметры автоматически: "Брусок 50×50×3000".

### Три типа товара
Тип определяет:
- какие поля нужны при добавлении в каталог
- как вводится остаток при инвентаризации

| Тип | Ввод остатка | Нужны размеры | Вместимость ячейки |
|-----|:---:|:---:|:---:|
| `unit` (штучный) | Числовой ввод (штук) | ✓ width × height × length | Расчётная или ручная |
| `bulk` (навалом) | Слайдер (% наполненности) | — | Ручная (пачки) |
| `round` (круглый) | Числовой ввод (штук) | ✓ diameter × length | Ручная (штук) |

Тип задаёт администратор при добавлении товара.

### Вместимость — настройка ячейки, не каталога
Каталог хранит размеры товара. Вместимость считается когда товар назначается на ячейку (S03). Для `unit` вместимость рассчитывается из размеров ячейки и товара. Для `round` и `bulk` вводится вручную. Для любого типа возможно ручное переопределение.

### pack_size — у всех типов
Брусок 50×50 → 4 штуки в пачке. Штапик навалом → 100 штук в пачке. Труба → N штук в пачке. В заявке дефицит показывается в штуках и пачках: "не хватает 150 шт = 2 пачки".

**Для типа `bulk`:** `pack_size` используется только для отображения ("X пачек · Y шт" в чеклисте). Расчёт дефицита ведётся напрямую в пачках, без деления на `pack_size` — детали в S05.

### Материал = группа для фильтрации и визуализации
Дерево, Пластик, Металл + кастомные. Управляемый справочник с именем и цветом. Цвет используется на схеме стеллажа. Более детальная фильтрация (бруски, плинтусы) не нужна.

## Модель данных

### Product

| Поле | Тип | Обяз. | Применяется для | Описание |
|------|-----|:---:|:---:|---------|
| `id` | uuid | ✓ | все | |
| `name` | string | ✓ | все | "Брусок", "Труба ПВХ", "Штапик" |
| `type` | enum | ✓ | все | `unit` \| `bulk` \| `round` |
| `material_id` | FK → Material | ✓ | все | Группа для фильтрации и цвета |
| `pack_size` | integer > 0 | ✓ | все | Штук в пачке |
| `width_mm` | integer > 0 | ✓ | `unit` | Ширина поперечного сечения |
| `height_mm` | integer > 0 | ✓ | `unit` | Высота поперечного сечения |
| `length_mm` | integer > 0 | ✓ | `unit`, `round` | Длина |
| `diameter_mm` | integer > 0 | ✓ | `round` | Внешний диаметр |
**Отображаемое имя:**
- `unit`: "Брусок 50×50×3000"
- `round`: "Труба ПВХ ⌀110×3000"
- `bulk`: "Штапик" (размеры не отображаются — нет смысла)

### Material

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | uuid | |
| `name` | string | "Дерево", "Пластик", "Металл" |
| `color` | hex string | Цвет на схеме стеллажа и в фильтрах |
| `is_custom` | boolean | false = предустановленный |

**Предустановленные:** Дерево, Пластик, Металл.
Администратор может добавлять кастомные.

## Что намеренно за бортом

| Что | Почему |
|-----|--------|
| Артикул / штрихкод | Нет существующих кодов, нет оборудования |
| Фото товара | Не нужно для v1 |
| Цена, поставщик | Приложение учётное, не торговое |
| Подкатегории (Бруски, Плинтусы) | Пользователь явно отказался |
| Размеры для `bulk` | Capacity — ручная, поштучный расчёт не нужен |
| Вместимость ячейки | Настройка ячейки (S03), не каталога |

## Зафиксировано для S03 (конфигурация ячейки)

**Первичный ввод при настройке ячейки — миллиметры** (измеряются рулеткой один раз). Приложение автоматически выводит количество штук. Причина: без точных мм расчёт поворота на 90° невозможен — приложение не знает остаток пространства после укладки целых рядов.


**Расчёт вместимости для `unit`:**
- По ширине: `floor(cell_width / product_width)`
- По высоте: `floor(cell_height / product_height)`
- Итого: ширина × высота
- Ручное переопределение перекрывает расчёт

**Поворот на 90°:**
- Если остаток пространства по ширине ≥ `product_height` → туда влезает один повёрнутый товар
- Повёрнутый товар ставится на край ячейки (левый или правый), не в середину
- Высота ряда при этом ограничивается максимальным из `product_height` (повёрнутый) и `product_height` (нормальный) — детали расчёта в S03

**Для `round` и `bulk`:** вместимость всегда вводится вручную при назначении товара на ячейку.

## Зафиксировано для S04–S05

**Отображение дефицита:**
- Первичное: штуки ("не хватает 30 шт")
- Вторичное: пачки ("2 пачки")

**Расчёт заявки:**
```
deficit_units  = capacity - current_stock   (штуки)
full_packs     = floor(deficit_units / pack_size)
```
- `full_packs ≥ 1` → позиция попадает в основную заявку с количеством `full_packs`
- `full_packs = 0` (дефицит есть, но меньше одной пачки) → **пограничная позиция**

**Пограничная позиция** — дефицит не набрал даже 1 полную пачку. Пример: дефицит 3 шт, пачка 4 шт. Отображается отдельно внизу заявки. Попадает в заявку только по усмотрению администратора (переключатель).

Дробный остаток сверх целых пачек (`deficit_units mod pack_size`) — не заказывается. Правило: лишнее не брать.
