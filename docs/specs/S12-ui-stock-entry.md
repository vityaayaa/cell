# S12 — UI: Внесение остатков

## Контекст использования

Сотрудник стоит перед стеллажом, телефон в одной руке. Перчатки при необходимости снимаются — задача требует точного ввода. Оптимизация под **точность**, не скорость.

---

## Навигация к форме ввода

Используется `ShelfGrid` из S11 в `mode="view"`. Логика тапа:

| Ячейка | Действие |
|--------|----------|
| Несплитованная (L1 или L2) | Тап → сразу открывается форма ввода |
| Сплитованная (L1) | Тап → переход на Level 2 → тап на суб-ячейку → форма |

---

## Прогресс обхода — фиксированная строка

Всегда видна над сеткой стеллажа, не скрывается при скролле:

```
Обход: 12 из 45 ✓                          [→ К заявке]
```

- **N из M**: M = листовые ячейки с назначенным товаром (S04). Ячейки без товара — не в счётчике.
- **→ К заявке**: переход к черновику заявки (S05).
- **Предупреждение**: если при нажатии "→ К заявке" внесено < 50% активных ячеек — показывается диалог: "Внесено только N из M ячеек. Продолжить к заявке?"  Если ≥ 50% — переход без предупреждения.

---

## Форма ввода: штучный товар (`unit`, `round`)

```
← A2(1,1)

Брусок 50×50×3000
Вместимость: 50 шт

[ __ ] шт

        [Сохранить]
```

| Элемент | Детали |
|---------|--------|
| Адрес ячейки | В header, кнопка ← возвращает к сетке |
| Название товара | 16px, `--foreground` |
| Вместимость | 14px, `--muted-foreground`. Показывается — это не предыдущий остаток, это физический размер ячейки |
| Поле ввода | `inputmode="numeric"`, шрифт ≥ 16px (иначе iOS Safari автозумит). Пустое при открытии — слепой пересчёт (S04) |
| Кнопка [Сохранить] | 56px высота, `--primary` (оранжевый). Неактивна если поле пустое или значение > вместимости |

**Валидация:**
- Остаток > вместимость → поле подсвечивается `--destructive`, кнопка заблокирована, сообщение: "Максимум N шт"
- Пустое поле → кнопка заблокирована, без сообщения (очевидно)

---

## Форма ввода: навалом (`bulk`)

```
← A2(1,2)

Штапик
Вместимость: 8 пачек

○────────────────○   ← слайдер 0–100%
        ≈ 4 пачки

        [Сохранить]
```

| Элемент | Детали |
|---------|--------|
| Слайдер | 0–100%, дискретных шагов нет — плавный |
| Подпись под слайдером | "≈ N пачек" — обновляется в реальном времени при движении. Значение = `round(fill_percent / 100 × capacity_packs)` (S04) |
| Вместимость | 14px, muted — для контекста |
| [Сохранить] | Активна всегда (0% — тоже валидное значение) |

Слайдер по умолчанию в положении 0. При открытии повторно — также 0 (слепой пересчёт).

---

## После сохранения

1. Запись идёт в Supabase (онлайн) или офлайн-очередь (S09).
2. UI обновляется оптимистично (S10).
3. Toast-уведомление:

```
✓ Внесено: 49 из 50 шт
▓▓▓▓▓▓▓▓▒▒▒▒▒▒  ← убывающий прогресс-бар
```

Для навалом: "✓ Внесено: ≈ 4 из 8 пачек (50%)".

4. Toast автозакрывается (~2 сек), форма закрывается, возврат к сетке.
5. Ячейка в сетке обновляется до состояния "✓ Внесено сегодня" (зелёный `#10B981`).

**Повторный ввод:** разрешён (S04). Сотрудник тапает на ячейку снова → пустое поле → вводит → перезаписывает.

## Визуальные состояния ячейки при обходе

Из S11 — дублируем для полноты:

| Состояние | Отображение |
|-----------|-------------|
| Внесено в текущем обходе | "✓ Внесено сегодня", цвет `#10B981` |
| Активный обход, не внесено | Дата последней записи (muted) |
| Нет активного обхода | Дата последней записи |
| Никогда не вносилось | "Не вносилось" (muted) |

---

## Realtime: изменение товара при открытой форме

Если во время открытой формы ввода остатка приходит Realtime-событие об изменении размеров товара, назначенного на эту ячейку (или о его удалении из каталога) — форма закрывается автоматически. Toast: "Параметры ячейки изменились. Откройте снова." Незаполненные данные не сохраняются.

---

## Переход к заявке

Кнопка "→ К заявке" в строке прогресса доступна в любой момент обхода — обход не нужно "завершать" явно (S04).

При нажатии:
- Если внесено ≥ 50% активных ячеек → сразу открывается черновик заявки (S05, фаза `ordering`)
- Если < 50% → диалог предупреждения, после подтверждения — то же самое

Нет промежуточного экрана-итога — сотрудник сам решает когда готов.

---

## Принятые решения

| Решение | Суть | Почему |
|---------|------|--------|
| Слайдер для `bulk` | Плавный 0–100% | Сотрудник оценивает наполненность визуально, не считает пачки. Приближённая оценка — норма для навалом |
| Кнопки 0/¼/½/¾/Полная — отклонены | Слайдер | Bulk — визуальная оценка, не дискретный выбор |
| Вместимость показывается в форме | Да | Не нарушает слепой пересчёт (это не предыдущий остаток). Нужна для контекста и для понимания тоста "X из X" |
| Toast с прогресс-баром | Убывающая полоска | Глобальное правило (S00): пользователь видит когда уведомление пропадёт |
| Автозакрытие toast | ~2 сек | Оптимизация под точность: экран стоит достаточно чтобы заметить опечатку в числе |
| Прогресс обхода — фиксированная строка | "N из M ✓ + → К заявке" | Ориентир + прямой доступ к заявке без поиска кнопки |
| Переход к заявке — напрямую | Без промежуточного экрана | Сотрудник сам решает когда готов; лишний экран замедляет |
| Предупреждение при < 50% | Диалог | Защита от случайного нажатия в середине обхода |

---

## Что намеренно за бортом

| Что | Почему |
|-----|--------|
| Показ предыдущего остатка | Слепой пересчёт (S04) |
| Кнопки ¼/½/¾ для bulk | Слайдер лучше для визуальной оценки |
| Фото ячейки | Не нужно в v1 |
| Экран-итог после обхода | Прямой переход к заявке достаточен |
| Таймер или подсказка "осталось N ячеек" | Счётчик в строке прогресса покрывает это |
