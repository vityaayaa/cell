# S11 — UI: Конструктор стеллажа

## Контекст использования

- **Устройство:** только телефон — admin и employee оба работают с телефона
- **Частота:** admin настраивает стеллаж ~раз в полгода; employee использует ежедневно в режиме обхода
- **Один экран, два режима** — компонент `ShelfGrid` с пропом `mode: "edit" | "view"`. Визуальная структура идентична; разница только в действии при тапе на ячейку

---

## Навигация: два уровня

### Уровень 1 — Обзор стеллажа (N×M базовых ячеек)

Весь стеллаж на одном экране с **двунаправленным скроллом** (X + Y).

**Минимальная ширина ячейки: 64px** (touch-target из S00). Ячейки горизонтального формата. Стеллаж 6 столбцов × 64px = 384px — чуть шире экрана, нормально для горизонтального скролла.

### Уровень 2 — Детальная ячейка

Открывается при тапе на **сплитованную** базовую ячейку. Отображает всё BSP-дерево этой ячейки на одном полноэкранном виде.

- Максимум 2 уровня навигации — из Level 2 дальнейшего drill-down нет
- Несплитованные ячейки не открывают L2: тап → сразу действие

---

## Level 1: Ячейки

### Несплитованная ячейка

```
┌──────────────────────────────┐
│ A1                        ⚠  │  ← адрес (12px, muted) + слот флагов
│                              │
│  Брусок 50×50×3000           │  ← название товара (16px)
│                              │
│  3 дня назад                 │  ← дата последней записи (12px, muted)
└──────────────────────────────┘
```

| Элемент | Позиция | Условие показа |
|---------|---------|----------------|
| Адрес (A1) | top-left, 12px, `--muted-foreground` | всегда |
| Слот флагов | top-right | только если есть активный флаг |
| Название товара | centre, 16px | если назначен; иначе "нет товара" muted |
| Дата | bottom-left, 12px | дата последнего `StockEntry`; "Внесено сегодня ✓" если в текущем обходе |

**Фон:** `Material.color` с 10% opacity (tint). Без товара — `--muted`.

**Дата в зависимости от состояния:**
- Активный обход, ячейка уже внесена → `"Внесено сегодня ✓"` (зелёный `#10B981`)
- Активный обход, ячейка не внесена → `"3 дня назад"` (muted)
- Нет активного обхода → дата последнего `StockEntry`
- Никогда не вносилось → `"Не вносилось"` (muted)

### Сплитованная ячейка

```
┌──────────────────────────────┐
│ A2                        ⚠  │  ← адрес + слот флагов
│                              │
│  ● Брусок 50×50×3000         │  ← цветная точка материала + название
│  ● Труба ПВХ ⌀110×3000      │
│  ● Доска 20×100×6000         │
│                              │
│  1 из 3 ✓               →   │  ← прогресс bottom-left, стрелка bottom-right
└──────────────────────────────┘
```

**Фон:** `--card` (несколько материалов — нет единого tint).

**Точка ●:** `Material.color`, 8px круг перед каждым товаром.

**Прогресс "N из M ✓":** только в режиме **активного обхода**. Вне обхода — пусто.

**→ стрелка:** всегда (обозначает drill-down).

### Слот флагов (top-right)

Зарезервированное место. При нескольких флагах — показывается приоритетный, остальные скрыты до тапа.

| Флаг | Иконка Lucide | Цвет | Условие |
|------|--------------|------|---------|
| Требует проверки | `AlertTriangle` | `#F59E0B` amber | `needs_review = true` (S03) |
| Поворот отключён | `RotateCcwSquare` | `--muted-foreground` | `rotation_allowed = false` (S03) |
| Вместимость вручную | `Pencil` | `--muted-foreground` | `capacity_override != null` (S03) |

Приоритет: `⚠` → `⊘` → `✎`.

## Level 2: Детальная ячейка

**Header:** кнопка ← (назад к L1) + адрес базовой ячейки (A2).

**Контент:** пропорциональный рендер BSP-дерева. Доли 50/50 из S02 — точно отражают физическую реальность стеллажа. Реализация: рекурсивный CSS Grid.

**Каждая листовая суб-ячейка** показывает те же элементы, что несплитованная L1-ячейка: адрес, товар, дата, слот флагов.

**Пример рендера (V-сплит, затем H-сплит внутри правой половины):**

```
← A2

┌──────────────────┬────────────────────────┐
│ A2(1,1)       ⚠  │ A2(1,2)             ⚠  │
│                  ├───────────┬────────────┤
│ Брусок           │ A2(2,1)   │ A2(2,2)    │
│ 50×50×3000       │           │            │
│                  │ Доска     │ Плинтус    │
│ 3 дня назад      │ 20×100    │ 30×10      │
│                  │           │            │
│                  │ 5 дн.назад│ сегодня ✓  │
└──────────────────┴───────────┴────────────┘
```

Фон каждой суб-ячейки — tint её материала (10% opacity).

## Взаимодействия: edit-режим (admin)

### Тап на листовую ячейку → нижняя панель (bottom sheet)

**Без товара:**
```
[Назначить товар                       ]
[ Разделить → ]    [ Разделить ↓      ]
[Настройки ячейки                      ]
```

**С товаром:**
```
[Сменить товар                         ]
[Убрать товар                          ]  ← деструктивный, красный текст
────────────────────────────────────────
[ Разделить → ]    [ Разделить ↓      ]
[Объединить с соседом                  ]  ← только если есть BSP-сосед
[Настройки ячейки                      ]
```

"Объединить с соседом" показывает имя сосед-ячейки: "Объединить с A2(1,2)". Сосед всегда один (BSP). При наличии товара в соседе — предупреждение перед слиянием (S02).

### Тап на ⚠ флаг (edit-режим)

Модальное окно:
```
Требует проверки

[Пояснение из S03: что именно изменилось]

[Открыть настройки]    [Всё в порядке ✓]
```

### Тап на сплитованную базовую ячейку в L1 (edit)

```
[Открыть ячейку →                      ]
[Размеры базовой ячейки                ]
```

### Экран "Настройки ячейки"

```
← Настройки A2(1,1)

Вместимость
  Расчётная: 108 шт
  Переопределение: [              ]  ← numeric, пусто = по формуле

Поворот                                 ← только unit с width ≠ height (S03)
  [●──────────────────] Разрешён

Размеры ячейки                          ← только базовые (корневые) ячейки
  Ширина: [ 545 ] мм
  Высота:  [ 400 ] мм
```

## Управление стеллажом (⚙ в header)

Действия над стеллажом в целом — не над ячейкой:

```
[+ Добавить строку снизу               ]
[+ Добавить столбец справа             ]
────────────────────────────────────────
[Переименовать стеллаж                 ]
```

Только append — из S02. Адреса существующих ячеек не меняются.

## Начальная настройка стеллажа

При первом входе admin, стеллажа нет → экран создания:

```
Стеллаж ещё не настроен

Рядов:    [ 4 ]
Столбцов: [ 6 ]

[Создать →]
```

После создания — пустая сетка N×M. Каждая ячейка показывает подсказку к следующему шагу:

```
┌──────────────────────────────┐
│ A1                           │
│                              │
│  Задайте размеры             │  ← dim_required: tint нет, текст muted
│                              │
└──────────────────────────────┘
```

Тап → открывается поле ввода ширины и высоты в мм. Пока размеры не заданы, вместимость не рассчитывается и товар нельзя назначить.

## Технические решения

| Вопрос | Решение | Обоснование |
|--------|---------|-------------|
| Рендер BSP (L2) | CSS Grid, рекурсивный | Браузер поддерживает nested grid; пропорции через `grid-template-columns/rows` |
| Drag-and-drop | **Нет** | BSP строго 50/50 — drag бессмыслен и конфликтует со скроллом |
| Сторонняя grid-библиотека | **Нет** | react-grid-layout — для произвольного resize, не BSP |
| Скролл L1 | `overflow: auto` на контейнере | 2D scroll нативно, без зависимостей |
| Иконки | Lucide (из shadcn/ui) | Уже в стеке S10; дополнительных зависимостей не нужно |

---

## Принятые решения

| Решение | Суть | Почему |
|---------|------|--------|
| Два уровня навигации | L1 = обзор, L2 = BSP одной ячейки | Touch-targets: при глубоких сплитах ячейки на L1 слишком малы |
| Пропорциональный рендер BSP | Доли 50/50 из S02 | Точно отражает физику стеллажа; сотрудник сразу ориентируется |
| Кнопки вместо drag-and-drop | Нет drag для сплита/мержа | BSP 50/50 делает drag бессмысленным; кнопки понятнее модели |
| Один компонент — два режима | `mode: "edit" \| "view"` | Одинаковый визуальный язык; разные действия по тапу |
| Слот флагов top-right | Одно место для всех флагов | Консистентность для всех ячеек; легко добавить новые флаги |
| Материальный tint как фон | 10% opacity Material.color | Заменяет цветную полоску слева; создаёт карту материалов |
| Нет статусных цветов дефицита | Убраны из S00 | Цифры в заявке достаточны; упрощает дизайн |

---

## Что намеренно за бортом

| Что | Почему |
|-----|--------|
| Drag-and-drop сплит / resize | Бесполезен: деление всегда 50/50 (S02) |
| react-grid-layout | Для произвольного resize — не подходит BSP |
| Статусные цвета уровней дефицита | Удалены из S00 — цифры достаточны |
| Undo / redo | Из S02: только merge как откат сплита |
| Визуальные пропорции строк/столбцов на L1 | Из S02: сетка визуально равномерная |
| Вставка строки/столбца в середину | Из S02: только append с края |
| Третий уровень навигации | L2 показывает всё дерево на одном экране |
