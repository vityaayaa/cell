# S14 — UI: Общий UX и навигация

## Контекст

Финальная UI-сессия. Определяет полную карту экранов, навигационную структуру и системные состояния приложения.

## Навигационная структура

### Employee — стек без таббара

У employee одна задача — сессия. Линейный workflow не требует постоянного переключения между разделами.

```
╔══════════════════════════════════╗
║       Сессии (главный)           ║  ← всегда первый экран
║  (активные + история inline)     ║
╚══════════════╦═══════════════════╝
               ↓ Продолжить / Начать
╔══════════════╩═══════════════════╗
║   Стеллаж (view + прогресс)      ║
╚══════════════╦═══════════════════╝
               ↓ К заявке
╔══════════════╩═══════════════════╗
║       Черновик заявки            ║
╚══════════════╦═══════════════════╝
               ↓ Финализировать
╔══════════════╩═══════════════════╗
║            Чеклист               ║
╚══════════════════════════════════╝
```

Навигация назад — через header (←). ⚙️ (top-right) — bottom sheet: переключатель темы + кнопка выйти.

### Admin — 3 вкладки + ⚙️

```
┌──────────────┬──────────────┬──────────────┐  ⚙️
│   📋         │   📦         │   🏪         │
│  Сессии      │  Каталог     │  Стеллаж     │
│  (Tab 1)     │              │  (edit mode) │
└──────────────┴──────────────┴──────────────┘
```

- **Tab 1 Сессии**: то же что у employee + admin-полномочия (force abandon, агрегаты, Excel)
- **Tab 2 Каталог**: список товаров + управление материалами
- **Tab 3 Стеллаж**: всегда edit-режим (конфигурация ячеек, размеры, товары)
- **⚙️ icon** (top-right): открывает экран Настройки (Аккаунты → , Журнал действий → , Тема, Выйти)

Каждая вкладка поддерживает свой стек навигации. Сессионный стек (Стеллаж view → Черновик → Чеклист) живёт внутри Tab 1.

Admin делает обход — тапает "Начать обход" в Сессиях (Tab 1) → следует тем же стеком что у employee. Вкладка Стеллаж (Tab 3) — только для edit-режима конфигурации.

## Экран «Сессии» — единый для обеих ролей

Показывает **все сессии всех пользователей**: активные вверху, завершённые ниже (история inline). Обе роли видят одинаковое; admin видит дополнительные action-кнопки.

### Нет активных сессий + нет истории

```
┌──────────────────────────────────┐
│ CELL                         ⚙️  │
│                                  │
│        Обходов пока нет          │
│                                  │
│       [Начать обход →]           │
│                                  │
└──────────────────────────────────┘
```

### Есть активные и завершённые сессии

```
┌──────────────────────────────────┐
│ CELL                         ⚙️  │
├──────────────────────────────────┤
│ ┌──────────────────────────────┐ │
│ │ 🔄  Обход · 3 июня           │ │
│ │ 23 из 45 ✓                   │ │
│ │ [Продолжить →]               │ │
│ │                    Отменить  │ │  ← деструктивный, красный
│ └──────────────────────────────┘ │
│                                  │
│ ┌──────────────────────────────┐ │
│ │ 📋  Чеклист · 1 июня         │ │
│ │ Взято 5 из 8                 │ │
│ │ [Открыть →]                  │ │
│ │                    Отменить  │ │
│ └──────────────────────────────┘ │
│                                  │
│    [+ Начать новый обход]        │  ← внизу; заблокировано при активном sweeping/ordering
│                                  │
├─── История ──────────────────────┤
│                                  │
│  ✓ 4 июня   Завершена  12 позиций│  → тап → детали
│  ✓ 28 мая   Завершена   8 позиций│
│  ✗ 20 мая   Брошена     —        │
│                                  │
└──────────────────────────────────┘
```

**Кнопка "[+ Начать новый обход]" заблокирована** при активной сессии в `sweeping` или `ordering` — подпись "Сначала завершите текущий обход". Сессии в `fulfilling` новый обход не блокируют (S05).

**Иконка карточки по фазе:**

| Фаза | Иконка | Подпись |
|------|--------|---------|
| `sweeping` | 🔄 | "Обход" |
| `ordering` | 📝 | "Заявка: черновик" |
| `fulfilling` | 📋 | "Чеклист" |

**Кнопка "Отменить"** — деструктивная, красный текст, мелкая, внизу карточки. Подтверждение: "Сессия будет отменена. Введённые данные обхода сохранятся." Доступна для всех фаз. Для employee — только своя сессия; для admin — любая.

**Чужой активный обход** (для employee):

```
│ ┌──────────────────────────────┐ │
│ │ 🔄  Иван ведёт обход         │ │
│ │ Начат 2 часа назад · 15/45   │ │
│ └──────────────────────────────┘ │
```

Кнопка "Начать обход" заблокирована. Свои чеклисты в `fulfilling` доступны.

**Admin-only на карточке чужой сессии:**

```
│ ┌──────────────────────────────┐ │
│ │ 🔄  Иван ведёт обход         │ │
│ │ Начат 2 часа назад · 15/45   │ │
│ │              [Завершить ×]   │ │  ← только у admin
│ └──────────────────────────────┘ │
```

**Admin-only внизу экрана** (после истории):

```
│  [Агрегаты по товарам →]         │
│  [Экспорт Excel ↓]               │
```

## Онбординг и пустые состояния

### Первый запуск — первый admin

После создания аккаунта (S07) admin видит Tab 1 Сессии. Стеллаж не настроен — кнопка "Начать обход" заблокирована с подписью "Настройте стеллаж сначала". Тап → переход на Tab 3 Стеллаж, где показывается пустое состояние:

```
┌──────────────────────────────────┐
│ 🏪 Стеллаж          ✏️ Изменить  │
├──────────────────────────────────┤
│                                  │
│    Стеллаж ещё не настроен       │
│                                  │
│    [Настроить стеллаж →]         │
│                                  │
└──────────────────────────────────┘
```

### Пустые состояния остальных экранов

| Экран | Пустое состояние |
|-------|-----------------|
| Каталог (нет товаров) | "Каталог пуст. [+ Добавить товар]" |
| Сессии (нет сессий и истории) | "Обходов пока нет" + [Начать обход] |
| Журнал действий (нет записей) | "Действий пока нет" |
| Аккаунты (только один admin) | Список с одним admin + "[+ Добавить сотрудника]" |

### Первый запуск — employee

Employee входит когда стеллаж уже настроен admin-ом. Видит экран Сессий с пустым списком и кнопкой "Начать обход". Туториал не нужен — экран говорит сам за себя.

## Офлайн-индикатор

**Место:** header, рядом с ⚙️. Одинаков на всех экранах.

| Состояние | Отображение | Когда |
|-----------|------------|-------|
| Онлайн | — (ничего) | Норма |
| Восстановление | зелёная точка + "Снова онлайн", 3 сек | После офлайна |
| Офлайн (нет очереди) | серая точка + "Офлайн" | Нет связи |
| Офлайн (есть очередь) | серая точка + "Офлайн · N записей ожидают" | Есть несинхронизированные данные |
| Синхронизация | `↻` + "Синхронизация..." | Очередь уходит на сервер |

Серый цвет — не красный: потеря связи не ошибка, а условие среды.

## Системные состояния

### Loading

Skeleton-экраны вместо контента. Серые прямоугольники-заглушки вместо ячеек стеллажа и карточек сессий. Нет спиннера на весь экран.

### Ошибки

| Ситуация | Поведение |
|----------|-----------|
| Не сохранилось (онлайн, ошибка сервера) | Toast (destructive): "Не сохранилось. Попробуйте ещё раз." |
| Не сохранилось (офлайн) | Toast (info): "Сохранено на устройстве. Загрузится при подключении." |
| Ошибка синхронизации | Toast (destructive): "Не удалось синхронизировать. [Попробовать снова]" |
| Некорректный ввод | Inline: поле подсвечивается `--destructive`, сообщение под полем |

Модальных окон с ошибками нет. Работа не прерывается.

## Восстановление навигации и кнопка "Продолжить"

### Куда ведёт "Продолжить →"

Кнопка на карточке активной сессии ведёт **по статусу сессии**, а не по последнему экрану:

| Статус сессии | "Продолжить →" ведёт на |
|--------------|------------------------|
| `sweeping` | Стеллаж (view + прогресс обхода) |
| `ordering` | Черновик заявки |
| `fulfilling` | Чеклист |

Это единственный надёжный подход: навигационный стек браузера может быть потерян (OS убил процесс, перезагрузка), но статус сессии всегда есть в базе.

### Поведение при возврате в приложение

| Ситуация | Поведение |
|----------|-----------|
| Приложение в фоне (переключил музыку, ответил на звонок) | Стек сохранён — пользователь там же где был |
| OS убил процесс (нехватка памяти) | Стек потерян — открывается экран Сессий, затем "Продолжить" по статусу |
| Принудительно закрыл вкладку браузера | Стек потерян — то же что выше |

Пользователь **никогда не прерывается** при простом переключении между приложениями.

## Принятые решения

| Решение | Суть | Почему |
|---------|------|--------|
| Единый экран Сессии для обеих ролей | История inline, активные сессии вверху | Убирает конфликт S07 vs S14; обе роли видят общую картину |
| Employee без таббара | Стек навигации, главный экран = Сессии | Workflow линейный; таббар для независимых разделов, не этапов одного процесса |
| Admin — 3 вкладки + ⚙️ | Сессии / Каталог / Стеллаж | Admin управляет конфигурацией — разделы независимы и равнозначны |
| Сессионный стек в Tab 1 у admin | Начать обход → стек на Tab 1 | Admin может переключаться между вкладками и вернуться к своему чеклисту |
| Стеллаж Tab — только edit | View-режим в сессионном стеке | Два разных входа в ShelfGrid по mode; нет путаницы |
| Онлайн-статус скрыт | Показывается только при изменении | Нормальный режим не захламляет header |
| Серый = офлайн, не красный | Нейтральный цвет | Красный = ошибка; потеря связи — условие среды |
| Admin ⚙️ → экран Настройки | Не bottom sheet (слишком много пунктов) | Аккаунты и Журнал — полноэкранные разделы |
| Employee ⚙️ → bottom sheet | Всего 2 пункта (тема + выйти) | Bottom sheet достаточен |

## Что намеренно за бортом

| Что | Почему |
|-----|--------|
| Таббар у employee | Workflow линейный |
| История как отдельная вкладка | Интегрирована в экран Сессий |
| Отдельный раздел History у admin | Заменён единым экраном Сессий |
| Мастер первого запуска | Пустое состояние + CTA достаточно |
| Push-уведомления | Один магазин, синхронизация при открытом приложении |
| Dashboard со сводкой остатков | Нет запроса на аналитику реального времени |
| Автоматический переход при завершении сессии | Зафиксировано в S13: toast auto-dismiss, история в Сессиях |
