## Навигационная структура

### Employee — стек без таббара

У employee одна задача — сессия. Линейный workflow не требует постоянной навигации между разделами.

```
╔══════════════════════════════════╗
║    Менеджер сессий (главный)     ║  ← всегда первый экран
╚══════════════╦═══════════════════╝
               ↓ Продолжить / Начать
╔══════════════╩═══════════════════╗
║   Стеллаж (view + прогресс)      ║
╚══════════════╦═══════════════════╝
               ↓ К заявке (кнопка в прогресс-строке)
╔══════════════╩═══════════════════╗
║       Черновик заявки            ║
╚══════════════╦═══════════════════╝
               ↓ Финализировать
╔══════════════╩═══════════════════╗
║            Чеклист               ║
╚══════════════╦═══════════════════╝
               ↓ toast → Перейти к истории
╔══════════════╩═══════════════════╗
║       История сессии (детали)    ║
╚══════════════════════════════════╝
```

Навигация назад — через header (← к предыдущему экрану). Кнопки и переходы зафиксированы в S12/S13.

### Admin — 4 таба

```
┌──────────┬──────────┬──────────┬──────────┐
│    🏪    │    📦    │    📋    │    ⚙️    │
│ Стеллаж  │ Каталог  │ История  │Настройки │
└──────────┴──────────┴──────────┴──────────┘
```

Admin делает обход — переключается в view-режим в первой вкладке, дальше тот же стек что у employee. Менеджер сессий — overlay при попытке начать новый обход при уже активной сессии другого пользователя.

## Главный экран по ролям

### Employee — Менеджер сессий

Всегда первый экран при открытии приложения. Показывает только **активные** сессии (sweeping, ordering, fulfilling). Завершённые и отменённые — не показываются.

**Состояния экрана:**

**Нет активных сессий:**
```
┌──────────────────────────────────┐
│ CELL                         ⚙️  │
│                                  │
│                                  │
│        Последний обход           │
│        3 дня назад               │
│                                  │
│       [Начать обход →]           │
│                                  │
└──────────────────────────────────┘
```

**Одна и более активных сессий:**
```
┌──────────────────────────────────┐
│ CELL                         ⚙️  │
├──────────────────────────────────┤
│                                  │
│ ┌──────────────────────────────┐ │
│ │ 🔄  Обход · 3 июня           │ │
│ │ 23 из 45 ✓                   │ │
│ │ [Продолжить →]               │ │
│ │                    Отменить  │ │  ← красный, мелкий
│ └──────────────────────────────┘ │
│                                  │
│ ┌──────────────────────────────┐ │
│ │ 📋  Чеклист · 1 июня         │ │
│ │ Взято 5 из 8                 │ │
│ │ [Открыть →]                  │ │
│ │                    Отменить  │ │
│ └──────────────────────────────┘ │
│                                  │
│    [+ Начать новый обход]        │  ← внизу, заблокировано если уже есть sweeping-сессия
└──────────────────────────────────┘
```

**Кнопка внизу заблокирована** когда есть активная сессия в фазе `sweeping` или `ordering` — с подписью "Сначала завершите текущий обход". Сессии в `fulfilling` новый обход не блокируют (S05).

**Иконка карточки по фазе:**
| Фаза | Иконка | Подпись |
|------|--------|---------|
| `sweeping` | 🔄 | "Обход" |
| `ordering` | 📝 | "Заявка: черновик" |
| `fulfilling` | 📋 | "Чеклист" |

**Кнопка "Отменить" на карточке** — деструктивная, красный текст, мелкая, внизу карточки. Подтверждение: "Сессия будет отменена. Введённые данные обхода сохранятся." Сессия переходит в `abandoned`. Доступно для всех фаз.

**Чужой активный обход** (кто-то из команды ведёт):
```
│ ┌──────────────────────────────┐ │
│ │ 🔄  Иван ведёт обход         │ │
│ │ Начат 2 часа назад · 15/45   │ │
│ └──────────────────────────────┘ │
```
Кнопка "Начать обход" заблокирована. Свои открытые чеклисты всё равно доступны.

**⚙️ иконка** (top-right) — не отдельный экран, а bottom sheet: переключатель темы + кнопка выйти из аккаунта.

### Admin — вкладка Стеллаж

Первая вкладка. По умолчанию edit-режим. Переключатель "✏️ Изменить / 👁 Просмотр" в header.

## Онбординг и пустые состояния

### Первый запуск — первый admin

После создания аккаунта (S07) admin видит вкладку Стеллаж. Стеллаж не настроен:

```
┌──────────────────────────────────┐
│ 🏪 Стеллаж          ✏️ Изменить  │
├──────────────────────────────────┤
│                                  │
│                                  │
│    Стеллаж ещё не настроен       │
│                                  │
│    [Настроить стеллаж →]         │
│                                  │
└──────────────────────────────────┘
```

Тап → форма создания стеллажа (из S11): строки, столбцы → готово.

Мастер из нескольких шагов — не нужен. Admin разберётся через интерфейс.

### Пустые состояния остальных экранов

| Экран | Пустое состояние |
|-------|-----------------|
| Каталог (нет товаров) | "Каталог пуст. [+ Добавить товар]" |
| История (нет сессий) | "Завершённых сессий пока нет" (без CTA) |
| Журнал действий (нет записей) | "Действий пока нет" |
| Аккаунты (только один admin) | Список с одним admin + "[+ Добавить сотрудника]" |

### Первый запуск — employee

Employee входит когда стеллаж уже настроен adminом. Видит менеджер сессий с пустым списком и кнопкой "Начать обход". Туториал не нужен — экран говорит сам за себя.

### История у employee

Истории как отдельного раздела нет. Единственный доступ — toast при завершении сессии ("Все позиции отмечены · Перейти к истории →" из S13).

## Офлайн-индикатор

**Место:** header, слева от названия приложения (или справа — рядом с иконкой ⚙️). Один и тот же на всех экранах.

**Три состояния:**

| Состояние | Отображение | Когда |
|-----------|------------|-------|
| Онлайн | — (ничего не показывается) | Норма |
| Восстановление связи | зелёная точка + "Снова онлайн", 3 сек | После офлайна |
| Офлайн | серая точка + "Офлайн" | Нет связи |
| Синхронизация | `↻` крутящийся значок | Очередь уходит на сервер |

Онлайн — не показывается постоянно: это нормальное состояние. Никаких модальных окон при потере связи — работа продолжается, действия идут в очередь (S09).

Цвет офлайн — нейтральный серый, не красный. Красный сигнализирует об ошибке; потеря связи — не ошибка, это контекст среды.

## Системные состояния

### Loading (первое открытие / синхронизация)

Skeleton-экраны вместо контента пока данные грузятся с Supabase. Серые прямоугольники-заглушки вместо ячеек стеллажа и карточек сессий. Нет спиннера на весь экран — UI не блокируется. Данные маленькие (< 1 МБ, S09), скелетон виден доли секунды.

### Ошибки

| Ситуация | Поведение |
|----------|-----------|
| Не сохранилось (офлайн) | Toast (info): "Сохранено на устройстве. Загрузится при подключении." |
| Ошибка синхронизации | Toast (destructive): "Не удалось синхронизировать. [Попробовать снова]" |
| Некорректный ввод | Inline: поле подсвечивается `--destructive`, сообщение под полем (S12) |

Модальных окон с ошибками нет. Работа не прерывается.

### Settings для Admin (вкладка ⚙️)

```
┌──────────────────────────────────┐
│ ⚙️ Настройки                     │
├──────────────────────────────────┤
│  Аккаунты                    →   │
│  Журнал действий             →   │
├──────────────────────────────────┤
│  Тема                            │
│  ○ Светлая  ○ Тёмная  ○ OLED    │
├──────────────────────────────────┤
│  Иван Иванов · admin             │
│  [Выйти из аккаунта]             │
└──────────────────────────────────┘
```

Тема — инлайн-переключатель. Аккаунты и Журнал — отдельные экраны.

## Принятые решения

| Решение | Суть | Почему |
|---------|------|--------|
| Employee без таббара | Стек навигации, главный экран = менеджер сессий | Workflow линейный: обход → заявка → чеклист. Таббар для равнозначных разделов, не для этапов одного процесса |
| Admin — 4 таба | Стеллаж / Каталог / История / Настройки | Admin управляет конфигурацией — разделы равнозначны и независимы |
| Заявка и Чеклист — не отдельные вкладки | Часть сессионного стека | Открыть их без активной сессии невозможно — в таббаре это мёртвые точки |
| История только у admin в таббаре | Employee — через toast | Employee не нуждается в самостоятельном просмотре истории; его задача — текущий обход |
| "Начать обход" внизу списка сессий | Ниже существующих карточек | Текущая работа важнее создания новой; стандартный паттерн мобильных списков |
| Онлайн-статус скрыт | Показывается только при изменении состояния | Нормальный режим не должен захламлять header |
| Серый = офлайн, не красный | Нейтральный цвет | Красный = ошибка; потеря связи — условие среды, не сбой приложения |
| Онбординг = пустое состояние + CTA | Без мастера шагов | Admin разберётся через интерфейс; мастер избыточен для одноразового действия |
| Skeleton-экраны вместо спиннера | Не блокировать UI при загрузке | Данные < 1 МБ грузятся быстро; skeleton оставляет экран работоспособным |

---

## Что намеренно за бортом

| Что | Почему |
|-----|--------|
| Таббар у employee | Workflow линейный, не требует постоянного переключения разделов |
| История в навигации у employee | Доступна через toast; самостоятельный доступ не нужен |
| Мастер первого запуска | Пустое состояние + CTA достаточно для одноразовой настройки |
| Push-уведомления | Не нужны: один магазин, небольшая команда, синхронизация происходит при открытом приложении |
| Dashboard со сводкой остатков | Нет запроса на аналитику в реальном времени; история (S08) покрывает потребность |
| Автоматический переход при завершении сессии | Зафиксировано в S13: toast требует нажатия |
