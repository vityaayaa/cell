# S14 — UI: Общий UX и навигация

## Контекст

Финальная UI-сессия. Определяет полную карту экранов, навигационную структуру и системные состояния приложения.

## Навигационная структура

### Таббар: морфинг по режиму

Таббар всегда снизу. Состав зависит от режима — **домашний** или **сессионный**.

| Роль | Домашний режим | Сессионный режим |
|------|---------------|-----------------|
| Employee | `[Главная]` | `[Главная · Стеллаж · Заявка · Чеклист]` |
| Admin | `[Главная · Стеллаж⚙ · Каталог]` | `[Главная · Стеллаж · Заявка · Чеклист]` |

**Домашний режим** — пока пользователь находится на экране Главной (не вошёл в активную сессию).

**Сессионный режим** — после нажатия "Начать обход" или "Продолжить →". Переход на рабочий экран + морфинг таббара.

**Анимация морфинга:** сессионные вкладки выезжают справа, spring-анимация 250ms (S00 A1). При возврате — складываются влево. Не блокирует UI во время анимации.

---

### Домашний режим

**Employee на Главной:**
```
┌──────────────────────────────────┐
│ CELL           [offline?]    ⚙️  │
│                                  │
│  [активные сессии / история]     │
│                                  │
├──────────────────────────────────┤
│           Главная                │  ← единственная вкладка
└──────────────────────────────────┘
```

**Admin на Главной:**
```
┌──────────────────────────────────┐
│ CELL           [offline?]    ⚙️  │
│                                  │
│  [активные сессии / история]     │
│                                  │
├──────────────────────────────────┤
│  Главная   Стеллаж⚙   Каталог   │  ← 3 вкладки
└──────────────────────────────────┘
```

---

### Сессионный режим (обе роли одинаково)

```
┌──────────────────────────────────┐
│ CELL           [offline?]    ⚙️  │
│                                  │
│  [рабочий экран: стеллаж /       │
│   черновик заявки / чеклист]     │
│                                  │
├──────────────────────────────────┤
│ Главная  Стеллаж  Заявка Чеклист │
└──────────────────────────────────┘
```

Вкладки отражают прогресс и доступность фаз:

| Вкладка | Маршрут | Активна при | Серая (некликабельна) при |
|---------|---------|------------|--------------------------|
| Главная | `/app/home` | всегда | — |
| Стеллаж | `/app/shelf` | sweeping, ordering, fulfilling | — |
| Заявка | `/app/order` | ordering | sweeping; fulfilling (нельзя назад — S05) |
| Чеклист | `/app/checklist/:id` | fulfilling | sweeping, ordering |

---

### Переходы между режимами

**Домашний → Сессионный:**
- Тап "Начать обход" или "Продолжить →" на карточке сессии
- Переход на нужный экран по статусу + морфинг таббара в сессионный набор

**Сессионный → Домашний:**
- Тап на вкладку "Главная"
- Сессионные вкладки складываются → возврат к домашнему таббару
- Переход на `/app/home`
- **Статус сессии не меняется** — она остаётся активной

Вкладки Стеллаж/Заявка/Чеклист **не видны на экране Главной** (даже при наличии активной сессии). Повторный вход в сессию — через "Продолжить →" на карточке.

---

### Стрелка ← и статус сессии

`←` в шапке — **только навигация, никогда не меняет статус сессии**.

Статус сессии меняется исключительно явными кнопками:

| Действие | Переход статуса |
|----------|----------------|
| Кнопка "→ К заявке" (S12) | sweeping → ordering |
| Кнопка "Финализировать" (S13) | ordering → fulfilling |
| Кнопка "Отменить" на карточке | любой → abandoned |
| Все строки чеклиста отмечены (S06) | fulfilling → completed |

**Кнопка "← К обходу" в заявке — упразднена.** Вместо неё — вкладка "Стеллаж" в таббаре: навигирует на стеллаж без смены статуса.

---

### Настройки ⚙️

⚙️ всегда в правом верхнем углу шапки, на всех экранах.

- **Employee**: ⚙️ → bottom sheet (переключатель темы + "Выйти из аккаунта")
- **Admin**: ⚙️ → полноэкранный экран `/app/settings` (Аккаунты, Журнал действий, Тема, Выйти)

---

## Экран «Главная» — единый для обеих ролей

Показывает **все сессии всех пользователей**: активные вверху, завершённые ниже (история inline). Обе роли видят одинаковое; admin видит дополнительные action-кнопки.

### Нет активных сессий + нет истории

```
┌──────────────────────────────────┐
│ CELL                         ⚙️  │
│                                  │
│        Обходов пока нет          │
│                                  │
│       [Начать обход →]           │
│                                  │
└──────────────────────────────────┘
```

### Есть активные и завершённые сессии

```
┌──────────────────────────────────┐
│ CELL                         ⚙️  │
├──────────────────────────────────┤
│ ┌──────────────────────────────┐ │
│ │ 🔄  Обход · 3 июня           │ │
│ │ 23 из 45 ✓                   │ │
│ │ [Продолжить →]               │ │
│ │                    Отменить  │ │  ← деструктивный, красный
│ └──────────────────────────────┘ │
│                                  │
│ ┌──────────────────────────────┐ │
│ │ 📋  Чеклист · 1 июня         │ │
│ │ Взято 5 из 8                 │ │
│ │ [Продолжить →]               │ │
│ │                    Отменить  │ │
│ └──────────────────────────────┘ │
│                                  │
│    [+ Начать новый обход]        │  ← заблокировано при активном sweeping/ordering
│                                  │
├─── История ──────────────────────┤
│                                  │
│  ✓ 4 июня   Завершена  12 позиций│  → тап → детали
│  ✓ 28 мая   Завершена   8 позиций│
│  ✗ 20 мая   Брошена     —        │
│                                  │
└──────────────────────────────────┘
```

**Кнопка "[+ Начать новый обход]" заблокирована** при активной сессии в `sweeping` или `ordering` — подпись "Сначала завершите текущий обход". Сессии в `fulfilling` новый обход не блокируют (S05).

**Иконка карточки по фазе:**

| Фаза | Иконка | Подпись |
|------|--------|---------|
| `sweeping` | 🔄 | "Обход" |
| `ordering` | 📝 | "Заявка: черновик" |
| `fulfilling` | 📋 | "Чеклист" |

**Кнопка "Отменить"** — деструктивная, красный текст, мелкая, внизу карточки. Подтверждение: "Сессия будет отменена. Введённые данные обхода сохранятся." Доступна для всех фаз. Для employee — только своя сессия; для admin — любая.

**Чужой активный обход** (для employee):

```
│ ┌──────────────────────────────┐ │
│ │ 🔄  Иван ведёт обход         │ │
│ │ Начат 2 часа назад · 15/45   │ │
│ └──────────────────────────────┘ │
```

Кнопка "Начать обход" заблокирована. Свои чеклисты в `fulfilling` доступны.

**Admin-only на карточке чужой сессии:**

```
│ ┌──────────────────────────────┐ │
│ │ 🔄  Иван ведёт обход         │ │
│ │ Начат 2 часа назад · 15/45   │ │
│ │              [Завершить ×]   │ │  ← только у admin
│ └──────────────────────────────┘ │
```

**Admin-only внизу экрана** (после истории):

```
│  [Агрегаты по товарам →]         │
│  [Экспорт Excel ↓]               │
```

## Онбординг и пустые состояния

### Первый запуск — первый admin

После создания аккаунта admin видит экран Главной. Стеллаж не настроен — кнопка "Начать обход" заблокирована с подписью "Настройте стеллаж сначала". Тап → переход на вкладку "Стеллаж⚙", где показывается пустое состояние:

```
┌──────────────────────────────────┐
│ 🏪 Стеллаж                   ⚙️  │
├──────────────────────────────────┤
│                                  │
│    Стеллаж ещё не настроен       │
│                                  │
│    [Настроить стеллаж →]         │
│                                  │
└──────────────────────────────────┘
```

### Пустые состояния остальных экранов

| Экран | Пустое состояние |
|-------|-----------------|
| Каталог (нет товаров) | "Каталог пуст. [+ Добавить товар]" |
| Главная (нет сессий и истории) | "Обходов пока нет" + [Начать обход] |
| Журнал действий (нет записей) | "Действий пока нет" |
| Аккаунты (только один admin) | Список с одним admin + "[+ Добавить сотрудника]" |

### Первый запуск — employee

Employee входит когда стеллаж уже настроен admin-ом. Видит экран Главной с пустым списком и кнопкой "Начать обход". Туториал не нужен — экран говорит сам за себя.

## Офлайн-индикатор

**Место:** header, между названием CELL и ⚙️. Одинаков на всех экранах.

| Состояние | Отображение | Когда |
|-----------|------------|-------|
| Онлайн | — (ничего) | Норма |
| Восстановление | зелёная точка + "Снова онлайн", 3 сек | После офлайна |
| Офлайн (нет очереди) | серая точка + "Офлайн" | Нет связи |
| Офлайн (есть очередь) | серая точка + "Офлайн · N записей ожидают" | Есть несинхронизированные данные |
| Синхронизация | `↻` + "Синхронизация..." | Очередь уходит на сервер |

Серый цвет — не красный: потеря связи не ошибка, а условие среды.

## Системные состояния

### Loading

Skeleton-экраны вместо контента. Серые прямоугольники-заглушки вместо ячеек стеллажа и карточек сессий. Нет спиннера на весь экран.

### Ошибки

| Ситуация | Поведение |
|----------|-----------|
| Не сохранилось (онлайн, ошибка сервера) | Toast (destructive): "Не сохранилось. Попробуйте ещё раз." |
| Не сохранилось (офлайн) | Toast (info): "Сохранено на устройстве. Загрузится при подключении." |
| Ошибка синхронизации | Toast (destructive): "Не удалось синхронизировать. [Попробовать снова]" |
| Некорректный ввод | Inline: поле подсвечивается `--destructive`, сообщение под полем |

Модальных окон с ошибками нет. Работа не прерывается.

## Восстановление навигации и кнопка "Продолжить"

### Куда ведёт "Продолжить →"

Кнопка на карточке активной сессии ведёт **по статусу сессии в базе**:

| Статус сессии | "Продолжить →" ведёт на |
|--------------|------------------------|
| `sweeping` | Стеллаж (view + прогресс обхода) |
| `ordering` | Черновик заявки |
| `fulfilling` | Чеклист |

Это единственный надёжный подход: навигационный стек браузера может быть потерян (OS убил процесс, перезагрузка), но статус сессии всегда есть в базе.

После "Продолжить →" таббар морфится в сессионный режим.

### Поведение при возврате в приложение

| Ситуация | Поведение |
|----------|-----------|
| Приложение в фоне (переключил музыку, ответил на звонок) | Стек сохранён — пользователь там же где был |
| OS убил процесс (нехватка памяти) | Стек потерян — открывается Главная, "Продолжить" по статусу |
| Принудительно закрыл вкладку браузера | Стек потерян — то же что выше |

Пользователь **никогда не прерывается** при простом переключении между приложениями.

## Принятые решения

| Решение | Суть | Почему |
|---------|------|--------|
| Единый экран Главная для обеих ролей | История inline, активные сессии вверху | Обе роли видят общую картину; настройки доступны из одной точки |
| Морфинг таббара | Домашний → сессионный режим при входе в обход | Таббар отражает текущий контекст; не занимает место когда не нужен |
| Сессионный режим одинаков для employee и admin | Одни и те же 4 вкладки при активной сессии | Admin в обходе = employee; никакой разницы в UI |
| ← не меняет статус сессии | Навигация назад — только движение по экранам | Статус = явное бизнес-действие, не следствие навигации |
| Кнопка "← К обходу" упразднена | Заменена вкладкой Стеллаж в таббаре | Вкладка удобнее и не создаёт путаницы со статусом |
| Онлайн-статус скрыт | Показывается только при изменении | Нормальный режим не захламляет header |
| Серый = офлайн, не красный | Нейтральный цвет | Красный = ошибка; потеря связи — условие среды |
| Admin ⚙️ → экран Настройки | Не bottom sheet (слишком много пунктов) | Аккаунты и Журнал — полноэкранные разделы |
| Employee ⚙️ → bottom sheet | Всего 2 пункта (тема + выйти) | Bottom sheet достаточен |

## Что намеренно за бортом

| Что | Почему |
|-----|--------|
| Постоянный таббар у employee | Морфинг: таббар появляется только при активной сессии |
| История как отдельная вкладка | Интегрирована в экран Главной |
| Разный сессионный UI для admin и employee | В режиме обхода оба видят одно и то же |
| Мастер первого запуска | Пустое состояние + CTA достаточно |
| Push-уведомления | Один магазин, синхронизация при открытом приложении |
| Dashboard со сводкой остатков | Нет запроса на аналитику реального времени |
| Автоматический переход при завершении сессии | Зафиксировано в S13: toast auto-dismiss, история в Главной |
