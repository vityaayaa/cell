# S00 — Дизайн-система CELL

Единый визуальный язык приложения. Обязательно для сессий S11–S14 и всей реализации.  
Если что-то в S11–S14 противоречит этому файлу — назвать противоречие явно, не идти молча.

---

## Шрифт

**Inter** — один шрифт на всё приложение.

- Отличная кириллица из коробки
- Читается на любом размере и яркости
- Стандарт для shadcn/ui-проектов

Масштаб: 12 / 14 / 16 / 18 / 24 / 32px. Поля форм — минимум **16px**, иначе iOS Safari автозумит страницу при фокусе.

---

## Три темы

Переключение пользователем: **светлая / тёмная / OLED**.  
Реализация: CSS-переменные через Tailwind + shadcn/ui. Один набор токенов, три набора значений.

| Токен | Светлая | Тёмная | OLED |
|-------|---------|--------|------|
| `--background` | `#F8FAFC` | `#0F172A` | `#000000` |
| `--card` | `#FFFFFF` | `#1E293B` | `#0A0A0A` |
| `--foreground` | `#0F172A` | `#F8FAFC` | `#F8FAFC` |
| `--muted-foreground` | `#64748B` | `#94A3B8` | `#94A3B8` |
| `--muted` | `#F1F5F9` | `#1E293B` | `#111111` |
| `--border` | `#E2E8F0` | `#334155` | `#1A1A1A` |
| `--primary` | `#F97316` | `#FB923C` | `#FB923C` |
| `--primary-foreground` | `#FFFFFF` | `#FFFFFF` | `#FFFFFF` |
| `--destructive` | `#EF4444` | `#EF4444` | `#EF4444` |
| `--destructive-foreground` | `#FFFFFF` | `#FFFFFF` | `#FFFFFF` |
| `--ring` | `#F97316` | `#FB923C` | `#FB923C` |

**OLED-правило:** никаких белых вспышек при переходах между экранами — анимации тёмные насквозь. Фон `#000000`, карточки чуть светлее (`#0A0A0A`).

## Индикаторы состояния

Фиксированные цвета, одинаковые во всех трёх темах.

| Индикатор | Hex | Где используется |
|-----------|-----|-----------------|
| Прогресс обхода ✓ | `#10B981` | "Внесено сегодня ✓" в ячейке (S11/S12) |
| Флаг предупреждения ⚠ | `#F59E0B` | `needs_review` на ячейке (S03/S11) |
| Деструктивное действие | `#EF4444` | Кнопки удаления, ошибки валидации |

**Статусные цвета уровней дефицита (зелёный / жёлтый / красный по % вместимости) — не используются.**  
В заявке достаточно цифр (дефицит в штуках и пачках из S05).

Индикатор никогда не передаётся только цветом — рядом всегда иконка или текстовая подпись.

---

## Акцентный цвет — оранжевый

Оранжевый (`#F97316` / `#FB923C`) — основной цвет интерактивных элементов: кнопки, ссылки, активные вкладки, фокус-кольцо.  
Янтарный (`#F59E0B`) занят флагом предупреждения ⚠ — достаточно отличается от оранжевого визуально, конфликта нет.

## Размеры кнопок и тач-целей

Контекст: сотрудник стоит у стеллажа, телефон в одной руке, возможны перчатки.

| Тип элемента | Минимальная высота |
|---|---|
| Основное действие (Сохранить, Взял, Финализировать) | **56–64px** |
| Вторичное действие, навигация | **48px** |
| Промежуток между соседними кнопками | ≥ **8px** |
| Опасное действие (удалить, финализировать) | ≥ 48px, **не рядом** с частыми |

## Анимации (Motion v12, уже в стеке S10)

**Пять правил:**

1. **150–250мс** для большинства микровзаимодействий. Комплексные переходы — не дольше 350мс.
2. **Spring-кривая** — не linear, не cubic-bezier. Ощущение живого, не механического.
3. **Выход быстрее входа**: exit ~100мс, enter ~200–250мс. Создаёт ощущение отзывчивости.
4. **Каждая анимация объясняет изменение состояния**, не украшает. Нет анимации без смысла.
5. **`prefers-reduced-motion`** — если включён, анимации отключаются или заменяются мгновенным переходом.

**Конкретные кейсы приложения:**

| Событие | Анимация |
|---------|----------|
| Отметка позиции в чеклисте | fade + slide вниз, 150мс |
| Сохранение остатка | кнопка `scale 0.97 → 1.03 → 1.0`, spring, 200мс |
| Переход вперёд по навигации | slide: новый экран приходит справа |
| Переход назад | slide: текущий уходит вправо, предыдущий появляется слева |
| Ошибка валидации | shake (100мс, 2 цикла) |
| Offline-статус | тихий pulse на иконке статуса, не модальное окно |
| Появление карточки/записи | scale `0.95 → 1.0` + fade, 200мс |

**Запрещено:** анимировать `width` / `height` — только `transform` / `opacity`. Блокировать UI во время анимации.

## Иконки

Только **Lucide** (входит в shadcn/ui по умолчанию). SVG-иконки, не эмодзи.  
Один стиль по всему приложению: outline, толщина обводки 1.5px.  
Размеры: 16 / 20 / 24px — не смешивать произвольно на одном экране.

---

## Стиль компонентов

**Flat + Bold.** Без теней, без стеклянных эффектов, без декоративных градиентов.  
Разделение секций — бордером (`--border`) или отступом, не `box-shadow`.

| Элемент | Скругление |
|---------|-----------|
| Карточки, панели | `rounded-lg` (8px) |
| Кнопки, поля ввода | `rounded-md` (6px) |
| Бейджи, теги статуса | `rounded-full` |

---

## Что это значит для S11–S14

- При описании компонента → цвета берём из токенов этого файла, не придумываем новые.
- При описании индикатора состояния → используем цвета из раздела «Индикаторы состояния» выше.
- При описании анимации → формулируем через кейс (что происходит), а не через CSS.
- Вопрос "какой стиль/цвет?" — **закрыт этим файлом**. Не переоткрывать без явной причины.
